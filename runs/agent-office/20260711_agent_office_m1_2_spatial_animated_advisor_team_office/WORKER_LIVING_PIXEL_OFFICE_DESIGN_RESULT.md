## 4. Renderer technology decision

The selected primary candidate is:

```text
React 19.2.7 (already present)
@pixi/react 8.0.5 (proposed exact later prototype pin; not installed)
PixiJS 8.19.0 (proposed exact later prototype pin; not installed)
preferred backend: WebGL
bounded backend fallback: tested Canvas core subset
deterministic fallback: existing DOM static M1.2 surface
final compatibility fallback: unchanged M1 fixed-station scene
```

Direct imperative PixiJS 8 under React was evaluated and rejected as the
primary candidate because it would require a custom React-to-Pixi reconciliation
and lifecycle layer. DOM/SVG/CSS-only continuation was rejected as the primary
visual experience but retained as the complete semantic/static/detail layer.
Pixi accessibility overlays were evaluated and not selected; the design
requires one separate, always-mounted DOM semantic mirror driven by the same
immutable frame model as Pixi.

The design records direct official evidence for React effect/StrictMode
lifecycle, `@pixi/react` React 19/PixiJS 8 support, `Application`, `extend`,
`useTick`, Pixi async init/ticker/resize/accessibility/textures/AnimatedSprite/
cleanup, current package versions, renderer fallback, MDN context loss and
resize observation, Vite lazy chunks, and Playwright live browser recording.
Evidence was checked on `2026-07-12`; no dependency was downloaded or installed.
