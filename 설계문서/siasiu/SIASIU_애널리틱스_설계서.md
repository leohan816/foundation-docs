# SIASIU 애널리틱스 (집계 행동분석) 설계서 (정본)

> **버전 v0.1 · 2026-06-20 · 🌱 아이디어 단계 (계속 추가)**
> "한 사람을 감시"가 아니라 — **전체 고객의 행동을 집계해 *무엇이 작동하는지* 측정.** 배너·페이지·콘텐츠·가격을 데이터로 최적화하고, AI 자율 운영의 *눈*이 된다.
> **쇼핑몰 설계와는 다른 문제(별도).** 정합: [커머스·쇼핑몰 설계서], [가드레일 설계서], [COSMILE 연동 설계서].
>
> **변경이력**
> - **v0.1 (2026-06-20)** — 최초 골격. Leo: 전체 고객이 *어떤 페이지·배너를 잘 클릭하는지* 집계 통계로 측정(개인 추적 ❌). 배너 성과 등. 동적가격 시그널 공급원.

## 0. 한 줄
**전체(익명·집계) 행동으로 "무엇이 잘 되는지" 측정 → 배너·진열·콘텐츠·가격을 데이터로 개선.** 개인 감시가 아니라 통계.

## 1. 목적
- **배너·콘텐츠 성과:** 어떤 배너/카드/페이지가 잘 클릭되나? A/B 어느 쪽이 이기나?
- **퍼널:** 어디서 들어와(유입) → 어디를 보고 → 어디서 이탈하나?
- **가격 시그널(★커머스 연결):** *클릭율은 높은데 구매가 적다* 같은 신호 → 동적 가격 조절의 *이유* 공급.
- **진열 최적화:** 어떤 제품/순서/이미지가 반응이 좋나 → AI 자율 머천다이징의 근거.
- ※ **개인 1명을 분석하는 게 아니다.** 전체를 합산한 **통계.**

## 2. 수집 이벤트 (예시 — 계속 추가)
| 이벤트 | 담는 것 |
|---|---|
| `page_view` | 페이지 id, 진입 시각, 유입 경로(referrer) |
| `dwell` | 페이지 체류 시간 |
| `click` | 요소 id(배너/카드/버튼/탭), 위치, 페이지 |
| `scroll_depth` | 어디까지 봤나 |
| `search` | 검색어(집계용), 결과수, 클릭 여부 |
| `impression` | 무엇이 *노출*됐나(클릭율 분모 — 노출 대비 클릭) |
| `add_to_cart` / `purchase` | 제품, 가격, 전환 |
| `ab_exposure` | A/B 변형 id(어느 배너·제목·가격을 봤나) |
- **핵심 지표:** CTR(클릭/노출) · 전환율(구매/클릭) · 이탈률 · A/B 변형별 성과.

## 3. 집계 단위
- **익명·합산.** 세그먼트(날짜·유입·언어·디바이스·A/B 변형)별로 *모아서* 본다.
- 개인 식별 분석이 목적 아님 — **통계.** (개인화 추천은 메모리가 따로 담당, 이건 *전체 최적화*.)

## 4. ★ 닫힌 루프 (측정 → 자율 운영 → 재측정)
```
[측정] 애널리틱스(집계: CTR·전환·이탈·A/B)
   → [신호] "클릭↑ 구매↓" · "배너 B가 승" · "이 진열이 반응 좋음"
   → [자율] AI가 가격 조절(이유 있음) · 배너 승자 채택 · 진열·이미지 갱신   (커머스 설계서)
   → [재측정] 다시 애널리틱스로 효과 확인 → 반복
```
= 애널리틱스 = **AI 자율 상점의 눈.** 사람은 틀(가드레일)만, 최적화는 이 루프가.

## 5. 기술 (MVP → 확장)
- **MVP:** 프론트가 이벤트 발생 시 `POST /api/event`(작은 JSON) → 백엔드가 **로그 파일/DB에 append**(`data/events.jsonl`) → 집계 쿼리로 지표 산출. (의존성 0로 시작 가능.)
- **확장:** 이벤트 테이블(DB) + 집계 잡(배치) + **대시보드**(CTR·퍼널·A/B). 규모 크면 전용 분석 파이프.
- **A/B 인프라**(커머스 §6)와 한 몸: 변형 노출(`ab_exposure`) + 성과(전환)로 **자동 승자 판정**(통계적 유의).

## 6. 프라이버시 · 가드레일
- **집계만.** 민감정보(주민번호·정확 위치 등) 수집 ❌. **동의(PIPA)** 전제.
- **개인 추적과 분리** — 메모리(개인화)와 애널리틱스(전체 통계)는 *다른 목적·다른 저장소.*
- **COSMILE 분리** — COSMILE 행동·매출과 엮지 않음(독립 회계, 연동 설계서).
- 모든 수집 항목은 **화이트리스트**(필요한 이벤트만, default-deny).

## 7. 🌱 열린 질문 (계속 추가)
- 이벤트 스키마 확정(어떤 요소에 id를 박을지 — 배너·카드·탭 표준화).
- 봇·어뷰징 필터(거짓 클릭 제거).
- 실시간 vs 배치 집계 / 보존 기간.
- A/B 통계 유의성 기준·최소 표본·자동 채택 규칙.
- 세그먼트 정의(유입·언어·신규/재방문).
