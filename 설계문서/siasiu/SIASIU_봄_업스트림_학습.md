## 0. 핵심 메시지 (봄이 *당장* 점검할 것 3가지)
1. **🔴 치명 보안버그 — 정적 서빙 경로우회 API 키 누출.** 봄도 같은 정적 서버 패턴이면 *지금 즉시* 점검. (§2.1)
2. **가드레일·검증은 *fail-closed*** (파싱 실패·예외 시 *막는 쪽*으로). 봄이 fail-open이면 위험. (§2.2)
3. **안전 사실은 *구조·결정론*으로** (LLM 기억에만 의존 ❌). 회피성분·알레르기는 atom 정규화 + 비대칭 보존. (§2.3, §3.2)

## 1. 루프 엔지니어링 방법론 (어떻게 루프를 돌렸나)

**정의:** *의미 있는 코드 블록이 추가되면* 서브에이전트 루프를 한 번 돌린다 — **테스트 작성 → 버그·잔여 수정 → 적대(adversarial) 검증 → 재실행.** "실행 품질"을 반복 개선하는 closed-loop. (관행화: 메모리 [[siasiu-loop-test-practice]].)

**루프로 풀기 좋은 7조건 (Leo):** ①반복 수정 가능 ②성공 기준 명확 ③되돌리기 가능 ④범위 제한 ⑤검증기 존재 ⑥개선 방향 관찰 가능 ⑦가치판단보다 *실행 품질*. → 이 조건이면 LLM 루프가 사람 없이도 수렴.

**오늘 돌린 루프들:**
- **루프1 (유닛 테스트):** 모듈별 영구 테스트 **405개 신규**(brain 73·ssbrain 147·app 185, 의존성 0·표준 라이브러리). → *키 누출 보안버그를 여기서 적발.*
- **루프2 (행동 시뮬):** 상담 전 과정(인사·되묻기·답변·검색·CLIR·메모리)을 *실제 `brain.chat()`으로 시뮬* + LLM이 판정. **mutation 테스트**로 단언의 의미성 검증(일부러 깨서 테스트가 잡는지)·불변(안전 비대칭·무휴리스틱) 보존 확인.
- **루프3 (다국어 매트릭스):** 7언어 × 4과정(인사·채팅·전환·섞임무시) = **28 시나리오**를 반복. 23→25→27→robust로 *수렴 과정 자체가 근본 결함을 드러냄*(프롬프트 땜질의 fragile함 → 아키텍처 재설계).
- **워크플로(병렬 다중 에이전트):** 인용 검증→교정(Opus 적대 + WebSearch), i18n 4언어 번역+검수 — fan-out으로 동시 처리.

**메타 교훈 (봄도 채택):** *루프는 버그를 만드는 게 아니라 드러낸다.* 키 누출·인용 환각·언어 flakiness 전부 "한 번도 제대로 체크 안 한" 코드에서 루프가 끄집어냄. **봄도 의미 블록마다 루프테스트를 관행화하면 같은 류의 잠복 결함을 잡는다.**

## 2. 버그 발견·수정 (상세: 무엇 / 어디 / 발견경로 / 근본수정 / 봄 관련성)

### 2.1 🔴 치명 — 정적 서빙 경로우회로 *API 키 누출*
- **무엇:** `GET /assets/../.secrets/deepseek_key` → **200 + 진짜 DeepSeek 키 반환.** (`.secrets/`·`data/`·`*.py` 소스도 노출 가능했음.)
- **어디:** `app/server.py` `_static_allowed()` — 화이트리스트가 URL **첫 세그먼트만** 보고 `..` 정규화를 *안 함.*
- **발견:** 루프1 app 보안 하드체크(공격 URL 5종 자동 시도).
- **근본수정:** 화이트리스트 검사 *전에* `path = posixpath.normpath(unquote(path or ""))` (디코드+정규화 먼저). `import posixpath, unquote` 추가. 검증: 공격 URL 5종 전부 404, 하드체크 영구화. (커밋 `2e0f742`)
- **★봄 관련성:** **봄도 로컬/데모 정적 서버가 있으면 동일 취약.** *정규화→화이트리스트* 순서 + `.secrets`/소스 차단 하드테스트 즉시 도입 권장.

### 2.2 가드레일·출력검증 *fail-OPEN* → fail-CLOSED
- **무엇:** `verify_output`이 파싱 실패 시 `d.get("ok", True)`로 *통과* 기본값. `guardrail_classify` 예외 시 "on/none"(허용)으로 샘.
- **어디:** `app/brain.py` `verify_output`, `guardrail_classify`.
- **발견:** 코드리뷰(xhigh) — 안전 경로의 기본값 점검.
- **근본수정:** `d.get("ok")`(True 기본 제거)·`"ok" not in d → ok:False`·classify 예외 → "off"(보수적). *불확실하면 막는 쪽.* (커밋 `12e91e0`)
- **★봄 관련성:** 봄의 모든 안전·검증 게이트 — *예외/파싱실패 시 어느 쪽으로 기우나* 감사. 안전은 fail-closed가 정본.

### 2.3 메모리 엔진 구멍 (근본화)
- **무엇:** ①"2번 규칙"(반복 언급 시 승격)이 no-op ②value 정규화 없음(같은 뜻 다른 표기 분산) ③회피성분 value↔atom 불일치(매칭 실패).
- **어디:** `app/brain.py` 메모리 계층.
- **근본수정:** `SAFETY_TYPES`(CANON의 is_safety에서 파생)·`CONF_PROMOTE=0.60`·`_norm_value`(bridge atom 수렴 + 결정론)·`avoid_atoms`·`_is_stale`(안전은 decay 면제)·`normalize_type` default-deny·SQLite **WAL+busy_timeout**·init_db 마이그레이션. **mutation 테스트로 안전 비대칭 보존 확인.** (커밋 `9d30ca1`)
- **★봄 관련성:** 봄 메모리도 *value 정규화(표기 흔들림 수렴)* + *안전 사실은 decay 면제·결정론 매칭* 채택 가치. (봄 백서가 "안전은 LLM 기억이 아니라 구조"라 한 것의 *구현*.)

### 2.4 인용 환각 (서지 fabrication)
- **무엇:** 성분 ~24개 파일에 *지어낸 저자*·틀린 수치(글리세린 진실 역전 99.4%)·중복 atom(copper-peptide)·카테고리를 atom으로(pha).
- **어디:** `knowledge/ingredients/` (볼트 사본).
- **발견:** 울트라코드 *검증→교정* 워크플로(Opus 적대 + WebSearch 재확인) — 192개 중 ~17개 인용오류 적발.
- **근본수정:** 워크플로+직접수정 교정 + **인용 표준 3층 방어**(WebSearch 확인 후만 기재·검증앵커 PMID/고시번호가 정본·저자는 *확인된 것만*, 틀린저자<저자없음·Opus 적대검수 게이트). (커밋 `3b01879`, 메모리 [[foundation-citation-standard]])
- **★봄 관련성:** 봄 지식 인제스트에 *인용 표준 + Opus 검수 게이트* 도입. LLM이 실재 PMID에 *없는 저자를 붙이는* 패턴은 봄 자료에도 잠복 가능.

### 2.5 ssbrain 검색엔진 버그들
- **무엇/어디:** ①`ingest.py _parse_conc` — 농도가 None으로 유실(% 정규화 누락) ②`_pitch_blocks` — `#` 헤더에서 본문 잘림 ③`engine.py` 리랭커 재정렬 버그(점수 뒤집힘) ④`schema.py` dead `chunks_fts`.
- **근본수정:** 농도 % 정규화 저장·헤더 경계 분할·버그성 재정렬 제거·dead 스키마 제거.
- **★봄 관련성:** 봄 BomBrain이 원형 — *리랭커 재정렬*·*청크 경계 잘림*·*수치 파싱 유실*은 봄에도 있을 패턴. 대조 점검 권장.

### 2.6 휴리스틱 근본 제거 (production 0)
- **무엇:** 키워드/정규식 추측이 의도·의미를 *어림짐작* → 근본 원인 못 막고 샘.
- **어디:** `app/brain.py` — `_infer_scale`(정규식)·`_ingredient_risk`+`_NEG_*`(키워드 회피탐지)·`_kb_hit`/`_stub_extract`.
- **근본수정:** ①scale = LLM 선언 신뢰 + 화이트리스트 *검증만* ②회피 = LLM 추출 avoid_list만(키워드 삭제) ③LLM 없으면 *추측 안 하고 정직히 비움.* 재스캔 휴리스틱 0 확인. (커밋 `b130524`, 메모리 [[foundation-no-heuristics]])
- **★봄 관련성:** 봄에 키워드/정규식 의미추측이 있으면 → 구조/결정론·KB grounding·LLM 의미판단으로 대체. (*날짜→명절* 같은 **정확한 결정론**은 휴리스틱 아님 — OK.)

### 2.7 bridge 토크나이저 — 교차어 가짜 atom
- **무엇/어디:** `app/ssbrain/bridge.py`+`text.py` `surface_match` — 라틴 부분문자열 매칭이 *다른 언어의 가짜 atom*을 만듦.
- **근본수정:** 라틴은 단어경계(word-boundary) 매칭, CJK 부분문자열 한계는 *정직하게 문서화*(은폐 X). (커밋 `0273a14`)
- **★봄 관련성:** 봄 CLIR 다리도 교차어 surface 매칭이면 같은 가짜매칭 위험.

### 2.8 다국어 출력 flakiness (가장 깊은 학습 — §3.1)
- **무엇:** ①DeepSeek-zh가 *큰 한국어 근거*에 끌려 *한국어로* 답(중국어 요청인데) ②영어에 한국어 단어 섞이면 언어가 뒤집힘.
- **어디:** `app/brain.py` 언어 계층.
- **근본수정:** B 프롬프트 현지화(`_localize`)·출력 언어 **보장**(`_reply_in_lang` 결정론 검사 → 2단계 재생성)·`_script_dominant`(섞임 임계). 매트릭스 27~28/28. (커밋 `c43e805`·`1e322e3`·`af3e29e`)
- **★봄 관련성:** 봄이 다국어 서비스로 가면 *이 아키텍처 전체*가 핵심 (§3.1).

## 3. 아키텍처 학습 (봄이 채택할 가치)

### 3.1 ★모국어 환대 — "찾기는 언어무관, 답하기는 고객 언어 *보장*"
오늘의 가장 novel한 결과. (정본: `설계문서/SIASIU_다국어_환대_설계서.md` v0.3)
- **시스템 언어(설정·신뢰) vs 발화 언어(감지)** 구분 — 인사·UI·기본은 시스템언어, 명확한 전환만 미러. 라틴 SEA(vi/ms/en)는 스크립트로 구분 불가 → *시스템 언어가 1차 신호.*
- **CLIR (찾기):** 질의를 코퍼스 언어들로 번역 → 자료가 *어느 언어로 있든* 찾아 RRF 융합. **근거 CONTENT는 절대 번역 안 함**(농도·수치 정확/안전 보존).
- **B = 프롬프트 현지화 (답하기):** 정본 한국어 PERSONA/지시를 *타깃 언어로 번역·캐시* → 프롬프트 언어 = 출력 언어 → 모델이 자연히 따름. (A=근거 번역은 *기각* — 정확도/안전 위험. 사후번역도 기각 — 톤 손실.)
- **출력 언어 보장:** 결정론 스크립트 검사 → 2단계 재생성(근거유지 강제 → 페르소나만). *고객은 못 읽는 언어를 절대 안 봄.*
- **★봄 관련성:** 봄이 다국어/해외로 가면 이 4블록(시스템vs발화·CLIR·B현지화·출력보장)이 그대로 이식 대상. 특히 *근거는 원문, 답은 고객 언어*가 핵심.

### 3.2 안전 비대칭 (safety asymmetry)
회피/알레르기 같은 안전 사실은 *놓치는 비용 ≫ 과하게 막는 비용* → decay 면제·결정론 매칭·default-deny. (메모리 엔진 §2.3, 가드레일 allowlist.)

### 3.3 allowlist > blocklist (가드레일)
"하지 말 것"은 무한 → 통제 불가. 대신 *무해 영역을 정의*(공감·일반케어=engage). 위험 카테고리(종교·정치·젠더·의료단정)만 *유한 blocklist*로 무조건 차단. (정본: `설계문서/SIASIU_가드레일_설계서.md` v1.1)

### 3.4 불필요한 선택은 감춤 (UX)
언어 선택 *벽* 제거 → 기기 언어 자동 감지(글로벌 표준). 강제 선택보다 스마트 기본. (메모리 [[foundation-hide-unnecessary-choices]]) — *단, 정확도 걸린 선택(안전)은 예외.*
