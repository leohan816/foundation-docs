## 0. 북극성 — 무엇이 *놀라운가*
검색 속도가 아니다. **"이게 *나를 기억*하고, *정직한 진실*을 말하고, *내 언어*로, *내 여정*을 함께 간다"**는 순간. 브랜드 4축(기억·관계·근본·신뢰)을 *진짜로.* → **깊게 갈 곳(wow)과 그럭저럭 갈 곳(배관)을 구분한다. 고르게 L5는 그럭저럭이다.**

**아픈 진실:** 지금 wow는 *약속*이지 *현실*이 아니다 — "정직·정확한 근거 있는 조언"인데 *근거(성분 안전 DB)가 0행.* 차별점이 아직 안 켜졌다.

## 1. ★지금 스프린트 — wow를 *진짜로* (= 각 서브시스템 L3 완성)
**4가지. 전부 "새로 만들기"가 아니라 *"만든 걸 연결"* — 코드 작고 ROI 최대.** (4갈래 만장일치.)

| # | 무엇 | 차원 | 효과 = wow |
|---|---|---|---|
| **①** | **벡터 영속화** (9분→초) | 개발 | 모든 개발·데모·테스트·측정의 *받침*. `schema`에 embeddings 테이블(chunk_id PK·vec BLOB float32·model_name·dim), ingest 1회 계산, `vector.build`를 DB로드 분기. model/dim 불일치 시 자동 재임베딩. |
| **②** | **성분 197 → ingredients 적재** | 데이터 | *정직·정확 조언이 **진짜**로.* `ingest_ingredient_dir()` 추가(atom·name 3언어·conc·임신·근거)+본문 색인. **bridge glossary 11→197 자동**(레지스트리에서 *생성*, 손수 ❌)·그래프·농도안전 동시 활성. ★명명표준 게이트(ingredient_id=slug=파일명 100%·불일치 거부)·Opus 검수. |
| **③** | **메모리 도메인 분리** (상담/코칭) | 개발 | *관계 차별점.* `memory_fact`에 domain 컬럼(멱등 ALTER+backfill 'skincare'). **결정론 매핑**(fact_type/화면→domain, LLM 추측 ❌). ★안전사실(allergy·avoid·pregnancy)은 *domain 무관 전역 회상*(안전비대칭 불변). |
| **④** | **검색 품질 측정** (qrels/nDCG) | 데이터 | *"작동"→"정확".* 골드셋 50~100(ko/zh/en·고민·성분·안전·교차언어) → nDCG@5·Recall@20·MRR. RRF·그래프부스트·리랭커·그라운딩 floor를 *측정으로* 튜닝. **측정 없는 튜닝=휴리스틱.** |

→ **이 4개가 wow를 약속에서 현실로.** ①②는 반나절~며칠. *안전척추가 켜지는 순간 검색·상담 품질이 즉시 도약.*

## 2. L-by-L 경로 (2차원: 개발/코드 · 데이터/규모)

### 2A. 검색 (L2~3 → L5)
- **L3(완성):** 위 ①②④ + bridge 자동확장 + CJK 세그멘테이션 정리. *단일 브랜드 유지(정확히 1개 먼저).* SQLite·numpy 브루트포스(19352청크=수밀리초·충분).
- **L4(하드닝):** qrels 측정 기반 튜닝 + **수치 Grounding floor**(약매칭 환각 차단) + 조건부 리랭커 자동발동 규칙 + **멀티브랜드 2~5개**(같은 성분=같은 atom 정합 = 명명표준 시험대) + search_logs 배선. 로컬 OK.
- **L5(프로덕션·Phase 6):** 외부색인 **pgvector(HNSW)**(numpy→교체, vector.py 추상화가 디딤돌) · 임베딩 서비스 분리(GPU노드) · BomBrain *무상태 수평확장* · 캐싱(Redis·휘발성 제외) · 무중단 재색인.

### 2B. 메모리 (L3 → L5)
- **L3.5(완성):** 위 ③ 도메인 분리. *코칭이 핵심 리텐션인데 상담 기억과 섞이면 차별점 흐려짐.*
- **L4(하드닝):** **추출 골든셋**(frozen 입력→기대 사실; 안전 recall=100% 게이트·일반 precision 우선=틀린기억<기억없음) + **신원 바인딩**(server.py가 POST user_id 무인증 신뢰하는 구멍 차단 — Phase2 Google 로그인 활용) + 동시성 부하 테스트 + CONF_PROMOTE 경계 테스트.
- **L5(Phase 6):** SQLite→**MySQL**(NestJS 허브) · 멀티유저 테넌트 격리 · 파티셔닝 · PIPA/GDPR. *동시성은 이미 WAL+BEGIN IMMEDIATE로 견고 → MySQL 전환은 측정 트리거 도달 시.*

### 2C. 데이터관리 (L2 → L5)
- **L3(완성):** 위 ② 성분 적재 + **볼트↔DB 정합 검사**(파일수==행수 불변식) + ko 본문 최소 채움(그라운딩 답 내용).
- **L4(하드닝):** **명명/인용 린터 = 게이트**(Opus 검수를 *사람 의지 아닌 자동 게이트*로 — 휴리스틱·환각·중복atom 누적 차단) + 멀티브랜드 데이터 + 현지화 콘텐츠(한국어 먼저→카테고리별).
- **L5(Phase 6):** MySQL/Postgres 분리 · CDN(이미지·에셋) · **COSMILE 공유 동기화**(볼트=정본·증분) · 출력 채널 템플릿(상세·라이브·몰·B2B).

## 3. 스케일 아키텍처 (10k+ 동접 — Phase 6 · ★설계만, 빌드 미룸)
**4갈래 만장일치: 지금 스케일 인프라 전부 미룬다. 0 유저. "필요할 때 스케일."** 핵심 기술 결정(미리 *정하되* 안 만듦):
- **벡터스토어:** 현 규모=numpy 브루트포스 + sqlite BLOB 영속이 정답. L5 폴리글랏 시 **pgvector(HNSW)** — *MySQL에 벡터 욱여넣지 않음.* 전용 벡터DB(Qdrant/Milvus)는 수백만 벡터·ANN 지연 *실측 시만.*
- **스토어 분리:** **MySQL=트랜잭션**(커머스·기억) / **Postgres+pgvector=검색.** 역할 분리.
- **BomBrain = 무상태 검색 서비스**(사용자 상태는 NestJS/MySQL) → 이 경계가 수평확장 전제. *L4에서 코드에 새긴다.*
- **임베딩 서비스 분리:** 검색(저지연)과 임베딩(배치·GPU) 워크로드가 달라 분리(L5 직전). 임베딩 캐시(chunk_hash→vec).
- **멀티유저:** `user_id`는 **NestJS JWT 검증 후 신뢰값만** BomBrain에 전달. *클라 user_id 직접 신뢰 ❌(안전사실 교차오염 차단).*
- **LLM:** DeepSeek API + 큐(BullMQ) + 캐시로 시작 → 셀프호스팅(vLLM/TGI)은 *비용/한계 실측 시.* `_llm` 라우터 추상화는 싸니 *일찍* 넣어 스왑비용 최소화.

## 4. ★안 만드는 것 (조숙한 과설계 경계)
**지금 금지:** pgvector·전용 벡터DB·임베딩 분리·수평확장·Redis·MySQL·CDN·셀프호스팅·샤딩. → 전부 Phase 6.
**왜 안전한가:** 추상화가 *이미* 돼 있다 — `vector.py`(numpy=FAISS 동일결과)·`_llm` 라우터·*볼트=정본/색인=파생*. **나중 스왑이 싸다** → 지금 미뤄도 손해 없음. 지금은 *경험의 깊이로* 이긴다.

## 5. ★Claude 적극 의견 (요약)
1. **고르게 L5 ❌ → wow부터 깊게.** 안전척추(②)+기억(③)+모국어(이미 됨)에 몰아준다. 나머지는 "쓸 수 있을 만큼."
2. **§1 스프린트 4개가 단일 최대 ROI** — 이미 만든 해자(성분 197·5채널 엔진)를 *연결만* 하면 품질이 도약. 새 데이터 수집보다 *연결*이 먼저.
3. **스케일은 트랙션 *후*.** 수만 동접은 "놀란 사람이 몰린 *결과*"지 전제가 아니다. 첫 100명을 *깊이로* 놀래자.
4. **교차의존:** 멀티유저 *검색* 격리(L5) ↔ 메모리 *도메인/namespace* 분리(L3.5) — 따로 가면 회피필터 user격리가 샌다. *같은 Phase에 묶는다.*
5. **측정이 휴리스틱과 종이 한 장 차이를 가른다** — qrels/골든셋 없는 RRF·추출 튜닝은 "느낌"=휴리스틱. *측정 게이트로만 임계 변경(Leo 승인).*
6. **자동 게이트 > 사람 의지** — 명명/인용/검수를 CI 린터로 강제(오류 누적 차단). Leo 규율을 코드가 지키게.

## 6. 한 줄
