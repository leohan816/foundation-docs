## 0. 한 줄 / 철학
**★★ 우리는 *사람이 하는 커머스*가 아니다 — AI-네이티브 커머스다.** 제목·상세·가격·진열·**판매 피치(음성)**까지 **전부 AI가 만들고 운영**한다. 사람은 *틀(가드레일)만* 정한다. 사람이 셀러·카피라이터·머천다이저·쇼호스트를 하는 기존 커머스와는 **종류가 다른 카테고리.**

**검색을 위해 제목을 비트는 게 아니라(쿠팡 ❌), 사람을 위해 제목을 짓고(감성·니즈) — 찾기는 *의미*가 한다(Meaning Bridge).** 운영(진열·가격·순서·이미지)은 **AI가 자율**로, 사람은 *틀(가드레일)만* 정한다.

**★ 구조적 해자 (왜 아마존·쿠팡이 못 따라하나):** 그들은 **마켓플레이스 — 수천만 셀러가 각자** 리스팅·상세를 쓴다 → 품질·정직·일관성 통제 불가(키워드 떡칠·과장·중구난방). **우리는 모든 상세페이지를 *우리가* AI로 직접 만든다** → 일관된 품질·정직·니즈 맞춤·컴플라이언스. 그들은 *무한 SKU(선택)*에 최적화, 우리는 *큐레이션된 완벽함(신뢰)*에 최적화. **다른 카테고리(상담형 커머스)** 라서 그들의 모델로는 *구조적으로* 흉내 못 낸다.

## 0.5 ★★ 운영 모델 — 사람은 *3가지*만, 나머지는 전부 AI (북극성 · Leo 2026-06-21)
**목표: 처음부터 끝까지 AI가 구동하는 시스템. 사람은 *결과에만 책임*진다.** 쿠팡·아마존·타오바오는 *리뷰·추천·상세페이지*로 팔고 — 결국 **운영 인력**이 필요하다. **SIASIU는 *니즈를 정확히 알아서* 팔고, 운영 인력은 없다 — 앞으로도.** 운영을 AI로 못 하면 우리는 그냥 전통 이커머스다.
- **★ 그래서 SIASIU 쇼핑몰은 바닥부터 전부 새로 개발한다** — 기존 쇼핑몰 위에 못 얹는다(AI-네이티브로 처음부터 설계).
- **사람이 하는 일 = 단 3가지 (물리·관계의 환원 불가능):**
  1. **采购(소싱·매입)** — 단, **AI가 수요·재고를 예측**하고 **가격·이익을 자동 설정.** 매우 정밀. 사람은 최종 매입·책임.
  2. **品牌沟通(브랜드 소통)** — 브랜드에서 *제품·자료*를 받아온다. 사람 관계.
  3. **发货(배송)** — 브랜드 직배 또는 우리 3PL(제3자 물류).
- **나머지 一律 AI (사람 0):** 진열·제목·상세·가격조정·순서·피치·추천·검색·**고객상담(CS)**·구매후 가이드·**고객관리(CRM)**·재고예측·가격/이익 설정·마케팅·세그먼트 … **전부.** (기존 쇼핑몰의 MD·CS·CRM·마케팅 팀 = 전원 AI 대체.)
- **★ 전제조건 (Leo):** *"이렇게 못 하면 나는 시작 안 한다."* = 옵션이 아니라 **존재 이유.**
- **사람의 역할 = 결과 책임·승인** — AI가 실행, 사람이 책임진다. ([[plan-before-build]] 승인 원칙과 같은 결.)

## 1. AI 자율 머천다이징 (사람이 컨트롤하지 않는다)
- **첫 화면 = 고객 니즈 맞춤 자동 진열.** 일률적 베스트셀러가 아니라 *이 사람*에게 맞는 제품들.
- **채팅 진입 순간 → 고객 정보 총동원**(메모리: 피부타입·고민·회피성분·과거반응·구매·목표) → 제품과 **실시간 매칭.**
- **가격과 순서 = AI가 결정.** 사람이 수동으로 안 정함.
- **동적 가격 (★ 전체 동일가):** 가격은 **모든 고객에게 똑같다** — 개인별 차등 ❌. 단 **AI가 집계 시그널로 가격을 자율 조절**하고, **조절엔 반드시 이유**가 있어야 한다(설명가능). 예) *클릭율은 높은데 구매가 적다* → 가격이 장벽일 수 있음 → 인하 검토. (재고·전환·계절 등 다른 신호도.) = 수요·전환 반응형, **항공권식 시점 가격과 같은 결 — 개인 차별 아님.**
- **메인 이미지 = 지속 A/B 테스트.** 더 잘 반응하는 쪽으로 계속 진화.
- → **"손이 덜 가는 상점."** 사람은 규칙·한계만, 최적화는 기계가.

## 2. ★ 사람을 위한 제목 (anti-쿠팡)
- 제목 = **제품을 가장 잘 표현 + 고객 니즈를 반영하는 감성적·인간적 카피.** (검색어 욱여넣기 ❌)
- **브랜드명만 앞에** 붙인다. 성분명·키워드는 *제목에 없어도 됨.*
- 예) 고객이 **"피부민감" / "피부진정"** 으로 검색 → **우리 제목엔 그 단어가 어디에도 없음.** 대신 그 제품을 가장 잘 표현하고 *그 니즈를 어루만지는* 문구로 보여준다.
  - (쿠팡식) `[브랜드] 시카 진정 크림 50ml 민감성 피부 저자극 수딩` ← 키워드 나열
  - (우리) `[브랜드] 하루의 끝, 붉어진 피부가 가라앉는 시간` ← 사람을 위한 제목
- **"이게 더 인간적이지 않나?"** → 그렇다. 그리고 이건 *우리만 할 수 있다*(§3).

## 3. ★ 제목 ≠ 검색 (핵심 — Meaning Bridge가 가능케 함)
쿠팡이 제목에 키워드를 욱여넣는 이유 = **어휘 검색**이라 제목에 단어가 없으면 *못 찾기 때문.* 우리는 다르다:
- **찾기는 제목이 아니라 *제품의 의미·속성*으로 한다.** 각 제품에 **숨은 메타데이터**(성분·효과·적합 피부·해결하는 고민·니즈 태그)를 둔다 = **검색용.**
- 질문 "피부진정" → Meaning Bridge가 *의미*로 그 제품의 속성과 연결 → **제목에 단어가 없어도 정확히 찾힘.**
- **제목 = 표시용(사람) / 메타데이터 = 검색용(기계).** 둘을 분리. 사람은 아름다운 제목을, 기계는 정확한 매칭을.
- **★ 이건 해자(moat)다:** 어휘검색 쓰는 경쟁사는 *못 따라 한다* — 제목을 인간적으로 바꾸는 순간 *검색이 망가지니까.* 우리는 Meaning Bridge가 있어서 **인간적 제목 + 정확한 발견**을 동시에 가진다. (검색 설계의 직접적 사업 가치.)

## 3b. ★ 상세페이지 — 텍스트 우선 + 이미지 더보기 (2층)
쿠팡식 *이미지 스크롤 지옥* ❌. 상세페이지는 **두 층:**
1. **우리 텍스트 페이지 (기본·우선):** 우리가 *고객 니즈에 맞춰* 멋지게 쓴 **정확한 텍스트.** 이미지가 없어도 *고객이 확실히 원하는 내용*을 정확히 전달.
   - 담는 것(예, 영양제): **핵심 성분(용량)** · **적용 대상**("이런 분께") · **용법** · **주의사항(정직)** · **기능의학 관점.**
   - 근거 = Leo 제품 라이브러리(`Leo_Products/韩承浩产品库`) 포맷: 정확·기능의학·**정직**("효과 논란 있음", "치표불치본"까지 솔직). → 단 고객용은 더 **따뜻·감성적으로** 가공(기조).
   - 빠르고 · 접근성↑ · 신뢰↑. **이미지를 안 봐도 핵심 100% 전달.**
2. **브랜드 상세페이지 (이미지·온디맨드):** 브랜드가 준(또는 우리가 추후 가공한) 이미지 상세. **"이미지 더보기"** 로 원하면 *언제든.*

**★ 한 몸(one record):** 텍스트 페이지의 사실(성분·적용대상·주의·기능의학)은 **검색 메타데이터(§3)와 같은 데이터.** 제품 1개를 *한 번 풍부하게 정리*하면 — **검색 발견 · 인간적 제목 · 텍스트 상세**가 다 그 위에서 돈다. (제품 데이터 = 모든 것의 뼈대.)
**가드레일:** 효능·완치 단정 ❌ · 주의사항 정직 · 기능의학 관점(가드레일 설계서 정합).

## 3c. 제품 레코드 스키마 (one record — Leo 라이브러리에서 검증됨)
Leo의 제품 라이브러리(`韩承浩产品库`)가 **이미 증명한 필드 구조**를 채택:
```
id · brand · name · spec(규격) · ingredients(성분+용량) · ingTags[] · funcTags[]/funcCats[](기능 분류)
· suitableFor(★적용 대상 — "이런 분께") · dosage(용법) · precautions[](★주의·라벨 원문 경고 보존)
· fmNote(★기능의학 관점) · price
```
- **이 한 레코드가 공통 소스:** 검색 메타데이터(§3) + 인간적 제목 생성(§2) + 텍스트 상세(§3b) + 매칭(§4)이 *전부 이 위에서* 돈다.
- **검증된 섹션 순서(상세):**
  - 보건품: 가격 → 용법 → 성분 → **적용대상** → 기능 → 특징 → 결핍 → 셀링포인트 → 주의 → **기능의학 참고** → Q&A
  - 스킨케어: 가격 → 제품정보 → **적용대상** → 핵심특징 → 성분표 → 브랜드소개 →(소스)
- **★ 내부→고객 변환:** 라이브러리는 *내부용*(정확하지만 안 예쁨). 우리는 **같은 사실을 따뜻·감성·니즈맞춤으로** 가공 → *정확성은 유지, 아름다움을 입힌다.*
- **fmNote의 힘:** "효과 논란 있음", "치표불치본"까지 **정직**하게 — 이게 우리 신뢰 차별점(행동진실).

## 3d. ★★ AI 음성 판매원 — 개인화 음성 피치 (Leo 2026-06-20 · "상세페이지 안 봐도 사는 비법")
**비법:** AI가 *이 사람이 이 제품을 사야 하는 이유*를 **현지 언어 음성으로**, 아주 우수한 판매원처럼 읽어준다 — **고객마다 다르게.** 상세페이지를 안 봐도 *듣기만 하면* 산다.
- **왜 가능한가 (이미 다 만듦 — 이게 그 목적이었음):**
  1. **메모리** → 이 사람 니즈·피부·고민·과거 반응
  2. **제품 레코드(§3c)** → 성분·적용대상·기능의학
  3. **DeepSeek** → 둘을 엮어 *이 사람을 위한* 피치 생성 (가드레일·광고법 §6b 준수)
  4. **Fish TTS(언어별 보이스: 아이유/en/zh/ja/es)** → 현지 언어로 **음성**
  → 입력(메모리)이 사람마다 다르니 **피치도 자동으로 1인 1피치.**
- **★ 1:1 라이브커머스:** 라이브 호스트는 *수천 명에게 한 번* 판다. 우리는 **각자에게 다른 피치** → *모든 고객이 자기만의 전속 쇼호스트*를 갖는다. (Leo의 라이브커머스 실력을 AI로 1:1 스케일.)
- **텍스트·이미지는 보조:** 듣기 = 노력 0 · 감정 · 신뢰. 더 알고 싶으면 텍스트 상세(§3b) · 이미지 더보기.
- **★ 가드레일 = 신뢰의 핵심:** "우수한 판매원" ≠ 조작. 우리 브랜드는 신뢰 → 피치는 **정직한 추천**(안 맞으면 "이건 지금 안 맞아요"까지). *정직해서* 안 보고도 산다 — 목소리를 믿으니까. (효능 단정·과장 ❌.)

## 3e. 음성 블록 조합 (Leo 2026-06-20 · 설계 확정 · 미구현)
통짜 생성 대신 **음성 블록을 조합**한다 — 블록당 **1회만 생성·캐시**, 조합은 공짜. 블록 하나만 수정 가능, 일관성↑.
- **생성 = Fish API로 코드 자동 생성**(사람 녹음 ❌ → 실수·네이밍 문제 없음). 파일명: `{product}_{block}_{lang}.mp3`. **이름은 음성에 안 넣음**(버튼에 이미 있음 — 토큰·재사용성↑).
- **"듣기 싫으면 종료"는 음성 끝에 ❌**(어색) → `intro`에 부드럽게("편하게 들어보세요") + 종료는 **UI 버튼.** closing = 따뜻한 마무리.

**블록:**
- 공통: `intro`(오프닝+이게 뭔지+편하게 안내) · `function`(기능) · `howto`(사용법) · `closing`(마무리)
- 적합: `fit_wrinkle`(주름·탄력) · `fit_beginner`(입문자) · `fit_dull`(칙칙함)
- 주의: `caution_sensitive`(민감→격일·진정먼저) · `caution_pregnancy`(임신·수유)
- 부적합: `avoid_retinol`(자극 경험→권 안 함) · `alt_suggest`(더 순한 대안)

**조합 = 조건식 (적합 → 주의 → 기능 → 사용법 → 마무리):**
```
playlist = []
if 회피성분∋레티놀 or 레티놀_자극경험:                 # 부적합 — 판매 스킵
    playlist = [ intro, avoid_retinol, alt_suggest, closing ]
else:
    playlist += [ intro ]
    if 고민∋주름 or 고민∋탄력:   playlist += [ fit_wrinkle ]   # ① 적합
    if 안티에이징_입문 or 레티놀_처음: playlist += [ fit_beginner ]
    if 고민∋칙칙함:             playlist += [ fit_dull ]
    if 적합블록_없음:           playlist += [ fit_beginner ]    # 기본값
    if 피부타입∋민감 or 민감_신호: playlist += [ caution_sensitive ] # ② 주의
    if 임신 or 수유:            playlist += [ caution_pregnancy ]
    playlist += [ function, howto, closing ]                  # ③ 기능→사용법→마무리
# 블록 mp3를 순서대로 이어서 재생
```
→ 민감성엔 민감 블록, 주름엔 주름 블록 — **블록 몇 개로 1:1처럼.** 안전(회피)은 코드 백스톱 우선.

## 3f. 상세페이지 레이아웃 (위 → 아래)
1. **음성 버튼 (상단 고정 sticky)** — "{이름}님을 잘 아는 아이유가 들려드려요" · ▶ 시작/종료 토글 · 홈처럼 흐르는 색.
2. **주图 캐러셀** (아마존식 회전).
3. **제목 · 한줄 · 가격.**
4. **★ 함께 쓰면 좋은 제품 (조합 추천)** ← Leo 2026-06-20. 이 제품과 *궁합 좋은* 제품을 묶어 제안 + **분류**(루틴 단계/카테고리). 예: 레티놀 + 진정 토너 + 보습 크림 + 선크림(레티놀 짝꿍). AI가 메모리·궁합으로 자동 구성.
5. **텍스트 상세 (문자만)** — 이런 분께 / 핵심 성분 / 사용법 / 솔직한 장단점 / 기능의학 / 브랜드.
6. **★ 비슷한 제품 (하단) — 피부별 맞춤 추천** ← Leo 2026-06-20. 이 고객의 피부타입·고민에 맞는 유사/대안. (민감성엔 더 순한 레티놀, 입문자엔 저농도 등.) **회피성분은 코드로 제외.**

→ 4·6 모두 **AI 자율 머천다이징**(메모리 × 제품 매칭). 4 = 궁합·루틴, 6 = 유사·대안. 가드레일(회피 제외 · 효능단정 ❌) 적용.

## 3g. ★ 구매 후 개인화 사용 가이드 — MY 열람 (Leo 2026-06-21)
구매하면 끝이 아니다. **구매 후에도** 그 제품의 *개인화 사용 가이드*를 MY에서 **계속** 본다 — *배송 확인하러 올 때마다* 거기서 바로.
- **위치:** MY 구매내역/배송 확인. 주문을 볼 때 가이드가 **항상 같이.** (배송 체크 = 자연스러운 재방문 트리거 → 우리가 그 순간 가치를 준다.)
- **★ 상세페이지 복붙 ❌ → 가공된 가이드:** *주의사항 · 사용방법 · 기대효과* 만 정리. 더 읽기 쉽고 행동가능하게.
- **★ 한 제품 = 여러 버전 (모든 상황 대비):** 고객 상황별로 다른 가이드 — 민감성/일반 · 함량 내성 · 병용 성분 · 입문/숙련. 메모리·프로필로 *맞는 버전* 선택. (피치 §3d · 블록조합 §3e와 **동일 아키텍처** — 제품 × 상황 생성·캐시·가드레일.)
- **★ AI가 가드레일 안에서 생성·작성.** 정적 문서 ❌ — 매번 그 사람·그 상황에 맞게 *써내려간다.* (효능단정·과장 ❌ · 광고법 §6b.)
- **★ 꼭 이름을 부른다 (친절하게) = 우리 원칙.** ※ 유일한 예외 = 음성 피치(이름이 버튼에 있어 음성엔 안 넣음 §3e). **글 가이드는 이름 호명.**
- **생성 = 관계 엔진 · 재료 = 제품 레코드:** 데이터는 제품 레코드(§3c)에서, *생성·전달*은 **AI 관계 엔진**(메모리+상담+코칭). → "수동 진열"이 아니라 **AI 능동 애프터케어.**
- **메모리·코칭 연결:** 가이드 데이터를 시아가 기억 → 구매 후 **능동 리마인드**("레티놀은 주 2회부터요 🙂", "어제 따갑다 하셨으니 오늘은 쉬어요"). 살아있는 정보. (코칭 설계서 cross-link.)
- **PDF 다운로드 = optional · 나중.** (제품사용설명 PDF. **고객상담(CS)와 무관** — CS는 실시간 AI 트랜잭션 §가드레일11 / Phase 7B.)
- **CS와 구분:** 이건 *애프터케어(정보·코칭)* 지 *트랜잭션(주문/환불)* 이 아니다.

## 3h. ★ 고객관리 (CRM) = 전부 AI (Leo 2026-06-21)
기존 쇼핑몰의 **CS팀·CRM팀·계정매니저·마케팅팀**이 하던 일을 **전부 AI**가. 사람 0. (운영 모델 §0.5의 "나머지 一律 AI"의 핵심 축.)
- **평생 메모리 = CRM의 척추:** 이 사람을 *알고·기억하고·케어.* 정본: 메모리 설계서.
- **능동 케어:** 구매 후 사용 가이드(§3g) · 코칭 리마인드 · **재구매 타이밍** · **이탈 징후 감지** → AI가 메모리로 *알맞은 순간*에 (사람 캠페인 ❌).
- **세그먼트·라이프사이클:** 신규/재방문/휴면/VIP 구분·맞춤 — **AI 자동.**
- **고객상담(CS):** 주문/환불 트랜잭션 = 실시간 AI(가드레일 §11 / Phase 7B). 일반상담(뷰티/건강)과 분리·메모리 격리.
- **리뷰·피드백:** 수집·반영도 AI 루프 (애널리틱스 설계서 연동).
- **사람 개입 = 결과 책임/승인만.** 일상 CRM 운영은 사람이 안 한다.

**★ 빌드 방침 (Leo 2026-06-20):** 현재 `product.html`은 *PoC(음성 판매원 개념 증명)* — 버리고 **완전히 새로** 만든다. 실제 상세페이지 = **제품 레코드(§3c) 기반 단일 템플릿이 모든 제품을 렌더**(제품마다 하드코딩 ❌). 음성 블록(§3e)·조합 추천·피부별 유사(§3f)·텍스트 상세(§3b)가 *처음부터 한 몸*으로 설계. → 데이터(레코드) 넣으면 페이지·제목·검색·음성이 자동으로 나오는 구조.

## 4. 매칭 = 메모리 × 검색
- 입력: 고객 메모리(피부·고민·회피·반응·목표·구매) + 현재 맥락(채팅·계절·날씨↔피부).
- 처리: Meaning Bridge로 니즈 ↔ 제품 메타데이터 매칭 + 회피성분 **코드 하드체크 제외**(가드레일).
- 출력: *이 사람*을 위한 진열·순서. 근거 없으면 밀어붙이지 않음(추천 보류, 가드레일 §7).

## 5. 가드레일 안의 자율 (자율 ≠ 무제한)
"사람이 컨트롤 안 함" = *세부를 안 함*이지 *한계도 없음*이 아니다. AI는 **우리가 정한 틀 안에서 자유**(default-deny 철학):
- **가격 하한/상한** — 원가·마진 보호(밑지고 안 팖), 폭리 방지.
- **회피성분·알러지** = 추천 절대 제외(코드).
- **효능·완치 단정 ❌**(제목·카피도 가드레일 적용 — "100% 진정" 같은 표현 금지).
- **브랜드 안전** — 카피·이미지가 우리 톤(따뜻·신뢰) 벗어나지 않게.

## 6. 🌱 열린 질문 (나중에 — 계속 추가)
- **가격 조절 시그널·규칙:** 어떤 집계 신호로 올리고/내릴지(클릭율·전환율·장바구니 이탈·재고·계절) + 변동 폭·빈도 한도 + **이유 로깅**(설명가능성). ※ 개인별 차등은 ❌ 확정 — **전체 동일가.** (애널리틱스 설계서가 이 신호를 공급.)
- **A/B 테스트 인프라:** 이미지·제목·가격 실험 프레임(노출·전환 로깅·통계적 유의·자동 승자 채택).
- **콜드스타트:** 메모리 없는 신규 고객 첫 진열은 무엇 기준으로? (베스트·계절·온보딩 입력)
- **제품 메타데이터 스키마:** 성분·효과·적합피부·니즈태그·금기 — 입력은 누가/어떻게(수기 vs 자동 추출)?
- **재고·가격의 휘발성:** 실시간만, 기억/지식에 안 박음(CLAUDE §0).
- **설명가능성:** "왜 이 가격/이 순서?"를 우리가 감사할 수 있게 로깅.
- **COSMILE 제품 vs SIASIU 자체 판매 분리**(독립 회계 — COSMILE 연동 설계서).
