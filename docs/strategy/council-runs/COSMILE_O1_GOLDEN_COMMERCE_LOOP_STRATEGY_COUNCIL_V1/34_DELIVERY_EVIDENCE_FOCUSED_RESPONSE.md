MD-04_POSITION: ACCEPT_WITH_NARROWING
MD-04_REASONING:
  Six statuses are justified because active work, external wait, timebox-closed missing evidence, and an affirmative stop are operationally different. Removing `EVIDENCE_INCOMPLETE` would leave expired internal work mislabeled as still active or as a substantive `HOLD`; replacing `PASS` with multiple claim-specific PASS variants would duplicate the mandatory claim-ceiling field and inflate the status vocabulary. The narrowing defines each status and compresses the record into grouped fields.
MD-04_EXACT_WORDING:
  Use exactly `NOT_STARTED | IN_PROGRESS_WITHIN_AUTHORITY | EXTERNAL_PENDING | EVIDENCE_INCOMPLETE | PASS | HOLD`.

  - `NOT_STARTED`: no authorized evidence work has begun.
  - `IN_PROGRESS_WITHIN_AUTHORITY`: authorized work is actively proceeding inside the current evidence ceiling and timebox.
  - `EXTERNAL_PENDING`: an identified external fact, vendor, counsel, or approval dependency prevents the claim from closing; useful in-authority outputs are frozen and the result returns with owner and reforecast, without automatic extension.
  - `EVIDENCE_INCOMPLETE`: the result or timebox has closed without all required internal evidence; it is not still active and cannot advance.
  - `PASS`: every predicate for the declared claim ceiling is evidenced and reviewed; it grants no further authority.
  - `HOLD`: evidence, a failed predicate, conflicting authority, or an explicit stop condition requires progression to stop and return for decision.

  Every result uses one compact record with: `RESULT_ID_AND_ENVIRONMENT`; `STATUS_AND_CLAIM_CEILING`; `EXACT_SUBJECT_CONTRACT_BUILD_CONFIG_POLICY_AND_EVIDENCE_PINS_AS_APPLICABLE`; `UNRESOLVED_IDS`; `ACCOUNTABLE_OWNER_AND_REVIEW_DISPOSITION`; `AUTHORIZATION_STATE_AND_ANY_SEPARATE_PIN`; and `STOP_OR_REFORECAST_RESULT_WHEN_NOT_PASS`. `AUTHORIZATION_STATE` defaults to `NOT_AUTHORIZED`; a later stage is authorized only by its own separate pin. `PASS_AT_DECLARED_CLAIM_CEILING` is not a separate status because `PASS` plus the mandatory claim ceiling already expresses that meaning. Detailed traces, test inventories, schedules, manifests, and runbooks remain linked evidence or later Advisor artifacts rather than duplicated claim records.
