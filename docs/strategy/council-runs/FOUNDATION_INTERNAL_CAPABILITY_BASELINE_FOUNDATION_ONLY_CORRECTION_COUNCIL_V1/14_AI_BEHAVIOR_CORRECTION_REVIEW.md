FINDINGS:

- FINDING_ID: AI-CORR-01
  SEVERITY: MATERIAL
  CLAIM: The contract names every relevant AI surface but does not require one per-behavior
    record that allocates responsibility among deterministic Foundation controls, model,
    provider, routing, prompt/context, fallback, memory, and tools.
  CONTRACT_EVIDENCE: The Current outcome envelope lists model/provider/routing/fallback/
    prompt/context/evaluation and deterministic logic/judgment/verdict/safety/evidence.
    P1 inventories those surfaces, and the common evidence record captures capability,
    method, environment, output, authority, failure, and claim ceiling. No required field
    identifies the behavior's input distribution, decision consequence, deterministic
    guard, exact model role, route/fallback oracle, tool allowlist/side effects, or model/
    agent stop limits.
  LENS_BASIS: Exact AI responsibility, model suitability, routing, fallback, prompt/context
    dependence, tool authority, cost, observability, and stopping behavior.
  RECOMMENDATION: Add a P1 `AI_BEHAVIOR_CONTRACT_AND_EVALUATION_MANIFEST` adjunct for each
    material AI path. It must record behavior/task and consequence; intended input classes
    and claimed languages/locales; deterministic preprocessing, policy, validation, and
    hard verdict constraints; model/provider/deployment identifier as actually exposed;
    prompt/template and context-assembly/truncation pins; generation settings; route reason;
    fallback/degraded oracle; memory read/write boundary; allowed tools and side effects;
    maximum steps, retries, time, tokens, cost, and tool calls; stop/abort outcome; scenario
    IDs, evaluator/oracle, repetition rationale, regression reference, and claim ceiling.
    P2 must record each run, including failures, actual route, retry/fallback, tool attempts,
    stop reason, latency, tokens, and observed or explicitly estimated cost. Exact counts and
    scenarios remain P1 evidence-dependent; a single stochastic run cannot support
    consistency and failures may not be rerun away.

- FINDING_ID: AI-CORR-02
  SEVERITY: MATERIAL
  CLAIM: The provenance-to-final-response probe class needs an explicit model-to-verdict
    transformation trace so retrieved evidence merely appearing in context is not mistaken
    for grounded use or verdict preservation.
  CONTRACT_EVIDENCE: P1 requires provenance and evidence reaching the final response, and
    P2 permits controlled synthetic conflict/ablation probes. The common evidence fields do
    not expressly separate the retrieved/ranked evidence, actual context sent, raw model
    output, parsed judgment, deterministic validation or override, final Foundation verdict,
    and downstream response.
  LENS_BASIS: Hallucination, evidence grounding, prompt/context dependence, overconfidence,
    judgment integrity, safety behavior, and Foundation verdict preservation.
  RECOMMENDATION: Extend each selected AI trace to pin retrieved evidence IDs/versions,
    rank and truncation; redacted actual model context; prompt/template hash; raw model
    output where safe; parsed structured judgment; deterministic validation, rejection, or
    override; final Foundation verdict and evidence; fallback event; and downstream
    preservation or explicit non-consumption. Use paired synthetic absence, conflict, stale/
    retracted, and controlled ablation cases only where P1 selects them. A citation's presence
    is not proof of causal use. Any unsupported material claim, unqualified response on
    missing/conflicting evidence, or weakening/reversal/omission of a pinned safety verdict
    is a preserved failed observation.

- FINDING_ID: AI-CORR-03
  SEVERITY: ADVISORY
  CLAIM: The probe classes are sufficient as a mandatory floor, but behavior-specific P1
    evaluation should add ambiguity, paraphrase, multilingual/context-limit, tool-denial,
    and repeated-failure cases only when the discovered behavior or support claim makes
    them decision-relevant.
  CONTRACT_EVIDENCE: P1 already includes ordinary, missing, conflicting, stale/retracted,
    unsafe/unsupported, unavailable, fallback/degraded, evidence-use, and recovery cases;
    P1 also prohibits deepening for target-architecture completeness.
  LENS_BASIS: Ambiguity, inconsistency, multilingual behavior, context dependence, refusal,
    tool authority, and proportional evaluation.
  RECOMMENDATION: Do not add a broad universal scenario catalogue. Require the P1 manifest
    to select ambiguity/clarification, semantic paraphrase, every explicitly claimed
    language/locale plus code-switch where material, context truncation, tool denial,
    repeated provider/tool failure, and cancellation/stop cases when those paths exist or
    support claims depend on them. If a competent oracle is unavailable, language or domain
    quality remains `UNVERIFIED` rather than being inferred from fluent output.
