## Recorded bypass / contradiction risks (evidence only; not patched)

- **R1** per-page auth (`layout.tsx:12` vs `guard.ts:5-8`) — a page omitting `requireConsoleUser` would render unguarded; all current pages call it.
- **R2** admin VIEW vs WRITE granularity — `console/admin` viewable by any ConsoleUser (incl viewer); writes owner/admin (`adminWrite.ts:5-7`).
- **R3 (CORE CONTRADICTION)** dual operator authority — O1 operator SCREENS gated by **ConsoleUser password session** (`requireConsoleUser`) while operator DATA/ACTIONS keyed to **Google-sub allowlist** (`o1OperatorForCustomer`/`makeO1StepUpVerifier`, role fixed `admin`). No unified operator-identity policy; a ConsoleUser absent from the Google-sub allowlist can open O1 operator screens. **Canonical policy left to Control — not decided here.**
- **R4** step-up nonce in-memory only (`o1Operator.ts:75-88`), dropped on restart; no DB grant/revocation model → DEFERRED durability.
- **R5** commerce dashboard mixes real events with `*_mock` alert rows (`commerceMetrics.ts:95-96`).
