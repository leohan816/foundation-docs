## Bounded findings

- Visual/token assets: global system Korean font stack; `#F6F5F3` canvas, `#1B1714` ink, `#6B645E` muted, `#F2622A` action, `#D94F1C` critical; rounded neutral cards and Tailwind utilities. Operator/admin surfaces use emoji/text and inline SVG only; no operator-specific image asset.
- Shell truth: `/console/**` is the permanent full-width control plane. `/o1/operator/**` falls through the 390×844 storefront frame, AppHeader, MallTabs, drawer/popup/cart chrome and is therefore transition evidence, not a permanent shell.
- Responsive truth: ConsoleShell changes from desktop sidebar to a four-link mobile top bar; wide tables scroll horizontally. ConsoleWorkspace keeps a 240px sidebar and hides artifacts below `lg` with no alternate. O1 remains one narrow column at all desktop widths.
- Accessibility strengths: O1 projection uses `dl`; action region exposes `aria-busy`; result uses `aria-live`; operator text inputs have accessible names; storefront icon actions have Korean labels.
- Accessibility gaps: Console login inputs have placeholder-only names and errors are not announced; most console/O1 controls lack a consistent visible `focus-visible` treatment or 40–44px target; nav has no `aria-current`; 10–11px muted text is pervasive; raw table overflow and hidden artifacts lose practical access at zoom/small widths.
- Korean quality: safety and economic-effect copy is unusually explicit and generally clear. Repair raw English enums/statuses, ISO timestamps, mixed `Dashboard/Operations/Write/Preview/step-up/mock`, and developer-facing endpoint/seed/v0 wording before a live default.
- Duplicate navigation: ConsoleNav desktop sections, ConsoleSubNav chips, dashboard action links, and ConsoleWorkspace’s separate dark sidebar expose overlapping routes with different labels and role rules; O1 adds a third navigation model through storefront chrome.
- Live-looking mock risk: commerce permits `production` filtering beside “mock 매출” and mock alert delivery; dry-run Foundation scores resemble live intelligence; Jobs and Admin mix real database rows with disabled or mock operational effects. Disclosures exist, but they are local rather than a persistent environment/capability boundary.
- Bounded critique: the strongest transition evidence is the allowlisted O1 active-request queue and fail-closed action panel. Only the independent Console persists; price, listing, event-planning, human-AI, and Agent Control Center placement seams remain deferred and nonfunctional until P2/later authority.
