## 3. Minimum schema decision (REQUIRED for M3–M5; NO WRITE here; BLOCKED on UD1–UD6)

Additive-only (`app/prisma/schema.prisma`), no edit/backfill of existing rows:
- `model OperatorPrincipal { id, subjectRef @unique, issuer, displayLabel(non-authority), state OperatorState @default(active), createdAt }` + `enum OperatorState { active suspended revoked }`.
- `model OperatorCredentialBinding { id, operatorPrincipalId FK→OperatorPrincipal, kind (console|google), consoleUserId FK?→ConsoleUser, subjectRef?, @@unique([kind,subjectRef]) }` — binds auth, confers no capability.
- `model CapabilityGrant { id, operatorPrincipalId FK, capability String(catalog), scope Json?(closed), source, grantedByPrincipalId FK?, reasonCategory, activatedAt, expiresAt?, state GrantState @default(active), revocationEvidence? }` + `enum GrantState { active suspended revoked expired }`; indexes `@@index([operatorPrincipalId,capability,state])`.
- `model StepUpFreshness { id, nonce @unique, action, scope Json?, operatorPrincipalId FK, reasonCategory, issuedAt, expiresAt, consumedAt? }`.
- Additive nullable `operatorPrincipalId String?` (+ FK/index) on the economic-audit row (`ConsoleAuditLog` or a new `OperatorActionAudit`; Control C5/UD6 to choose).
- Migration: forward = create tables/enums/columns; **down** = drop them; **no-backfill** = existing O1 operator continues via env-allowlist compatibility boundary (40 §5) and existing audit rows stay subject-null (truthfully unattributed, R4/C5); **deletion** = principals/grants transition to `revoked` state, never hard-delete (audit integrity). Disposable Postgres only.
- Why no smaller reuse is safe: `ConsoleUser/Session` = password identity, not a principal; env allowlist has no activation/expiry/revocation record; in-memory nonce is non-durable; `ConsoleAuditLog.userId` is a `ConsoleUser` FK so a Google-sub operator is un-attributable (31 C5). None can safely carry principal/grant/durable-freshness/subject-attribution.

## 4. OperatorPrincipal + Console credential binding (INV-0)

Provider-neutral `OperatorPrincipal` is the single authority root (40 §5, 31 C1). A Console credential/session may **authenticate** a human (via `OperatorCredentialBinding`) but **mints no O1 capability**; a `CapabilityGrant` is required and separate. Customer `AuthIdentity`/shopper session is **never** an operator credential or grant source — the current customer-derived Google-sub operator (`o1CommerceRuntime.ts:537-547`) is an explicit non-production compatibility boundary preserved until a separately reviewed migration (40 §5, 31 C1 UD1). Principal states `active|suspended|revoked`; all others deny. Display/email/session-id never authority keys.
