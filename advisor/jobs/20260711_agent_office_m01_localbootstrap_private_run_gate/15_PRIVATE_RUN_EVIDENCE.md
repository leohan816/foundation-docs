## 4. Crash, Stale Lock, and Stale Proof Recovery

The Advisor performed one bounded local crash simulation:

1. A second clean LocalBootstrap start created a new pending proof.
2. The exact listener process was force-stopped, leaving the writer lock and
   proof file as designed.
3. Immediate restart failed closed with `SECOND_WRITER_DETECTED` and opened no
   listener.
4. The authorized `WriterLock.recoverStale` primitive verified the dead process,
   preserved the stale lock plus recovery metadata in owner-only quarantine, and
   removed the active lock.
5. Restart then failed closed with `AUTH_PROVIDER_UNAVAILABLE` because the stale
   proof file remained; no listener opened and the proof was not overwritten.
6. With no process/listener/active writer, the owner removed the stale proof
   without reading or displaying it.
7. A fresh restart succeeded with a newly generated proof.

This proves both independent fail-closed layers and the explicit recovery route.
No automatic lock break, proof overwrite, or proof reuse occurred.
